## set

Un set est un ensemble non ordonnées d'objets uniques. Il est 
implémenté avec un table de hash, donc il est optimisé 
pour l'accés, l'insertion, l'effacement et le test d'appartenance. 
Il faut donc toujours les préférer à des listes lorsque l'on fait
un test d'appartenance (le coût de convertion est souvent négligeable
lorsque l'on fait beaucoup de tests d'appartenance). 

Comme le dictionnaire, un set ne peut accepter que des clefs
immuables (nombre, chaîne de caractères, tuple d'immuable).
Par contre, contrairement au dictionnaire, on ne peut pas stocker
de valeur associée à une clef. 

Les [sets] sont des objets mutables et il existe un type [frozenset]
qui a les mêmes caractéristiques qu'un set, mais qui est immuable. 

Ouvrons un terminal interactif pour jouer avec les set. 

[[
set mutable
frozenset immuable
]]
